*Source: https://forum.skyeco.com/t/blue-ad-recognition-submission/20915/208*

*Relates to: Spell recording (April 9, 2026) | Vote: **Unspecified***

- **Root cause** — Manual review used visual side-by-side comparison rather than exact-match / ctrl-f check from forum provenance post to executive sheet; the 7-of-8 digit match made the typo superficially plausible.
- **Process gap** — Provenance review should be a three-way match: forum post, executive sheet, and implemented code must all agree on the exact amount.
- **Automation gap** — GitHub Action 24250881578 flagged the Grove action as REQUIRES REVIEW and confirmed code/sheet match of 20,797,477 USDS, but did not surface an explicit forum-vs-sheet mismatch message.
- **Remediation** — Will require exact-match workflow for provenance checks; scripts to emit explicit mismatch messages rather than only REQUIRES REVIEW; acknowledges @cloaky for finding the discrepancy.
